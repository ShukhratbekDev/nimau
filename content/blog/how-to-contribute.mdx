# Loyihaga Qanday Hissa Qo'shish Mumkin

**"Nima U"** loyihasi ochiq manba hisoblanadi va texnik hamjamiyatni loyihaga hissa qo'shishga taklif etadi. Agar siz dasturiy ta'minotni yaxshilash, hujjatlarni to'ldirish yoki dizaynda yordam berishni istasangiz, sizning hissangiz qadrlanadi.

## Hissa Qo'shish Jarayoni

Loyihaga hissa qo'shish uchun quyidagi qadamlarni bajarishingiz mumkin:

1. **GitHub Repository Tashrif Buyurish**: Avvalo, [Nima U GitHub repository](https://github.com/Shukhratbek/nimau)-siga tashrif buyuring va uni `fork` qiling.
2. **Lokal Nusxani Yaratish**: Fork qilingan repository-ni o'z kompyuteringizga `clone` qiling.
3. **Yangi Branch Yaratish**: O'zgarishlarni asosiy branchga to'g'ridan-to'g'ri qo'shmaslik uchun yangi branch oching.
4. **O'zgarishlarni Amalga Oshirish**: Kerakli o'zgarishlarni amalga oshiring va kodni sinab ko'ring.
5. **Conventional Commits Qoidalarini Qo'llash**: O'zgarishlarni tavsiflovchi aniq va ma'noli commit xabarlarini yozishga harakat qiling. Bu haqida quyida batafsilroq.
6. **Pull Request Yuborish**: O'zgarishlarni asosiy repositoryga qo'shish uchun pull request yuboring.

## Conventional Commits

**[Conventional Commits](https://www.conventionalcommits.org)** â€” bu commit xabarlaringizni aniq va o'qiluvchan qilish uchun qoidalar to'plamidir. Bu o'zgarishlar tarixini tushunarli qilishda va avtomatik nashrlarni yaratishda yordam beradi.

### Conventional Commit Format

Har bir commit xabari quyidagi formatga rioya qilishi kerak:

```plaintext
<type>(<scope>): <short summary>
```

#### Misol:

```plaintext
feat(docs): add closure glossary to JS folder
```

Bu yerda:

- type: Commit turini ko'rsatadi (masalan, feat, fix).
- scope: O'zgarish ta'sir qilgan qism (masalan, docs, blog, api).
- short summary: O'zgarish haqida qisqa va aniq ma'lumot.
